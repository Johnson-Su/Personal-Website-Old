Backyard Hacks - Hackathon

Planty

Google Chrome extension that suggest ingredient substitutes with lower carbon footprints for online recipes

Devpost | Github

Problem

I want to substitute the high carbon footprint ingredients in my recipe with more ecological vegan alternatives.

Solution

An Google Chrome extension that will automatically suggest substitutes for high pollution ingredients when an online recipe is opened.

My Role:
Extension creation, Backend development, UI/UX Design

Tools used:
Javascript, HTML5, CSS3, Browserify, Firebase, node.js, Illustrator, Figma

Team:
Linda Ma, Amy Han, and Keke Zhang

Duration:
48 hours

You need to drive 63 miles to produce the same emissions as eating one kilogram of beef.

- EPA Guide to Passenger Vehicle Emissions

What it does

Planty is a Google Chrome extension that can be opened while browsing online recipes. It will automatically find ingredients with high carbon footprints and suggest a more environmentally friendly vegan substitute from a database. After a substitution is made Planty will calculate how much carbon the substitution saved.

What I did

I worked on the backend and created Javascript functions to read and write to a Firebase database in real time. Additionally I made functions to process the information pulled from the web scraper in order to create the substitutions and the carbon saving calculations with the information we had in our database.

Also, I created the UI for the extension using Figma and then converted it to HTML, CSS, and Javascript. I made the extension extremely easy to use (just one click) and also visually appealing at the same time.

Challenges & acheivements

In this project one of the main challenges was that I was creating an extension using tools that I had only learnt recently. This meant much of my time was spent on learning as there was very little information on how to create Chrome Extensions online. This really forced me to rely less upon tutorials to fix immediate issues and focus on understanding what went wrong myself. The restrictions that the extension had made it much harder since I had to deal with many asynchronous functions.

However, in the end I was able to grasp more complex ideas such as returning promises in order to process the information in order. I had achieved everything that I had set out to do and I was able to learn a lot more about how to incorporate databases, web scraping, and UI development and the impact of food on the environment.

My takeaways

By stepping outside of my comfort zone and choosing to use tools and concepts that were new to me I was able to really push myself to be more efficient with my work in order to accomplish what I desired within the timeframe. What this did for me was that I was able to improve my time management and problem solving alongside bolstering my leadership and communication skills as I worked with my amazing team to turn our thoughts into reality.

Besides improving my career skills, this project made me enjoy hackathons a lot more as I not only had fun with my team but also learnt more about the issue that I set out to solve too. Through compiling the environmental data I was really able to see how much of a difference simple choices can make even to the largest of issues.
